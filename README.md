# Playwright JavaScript Test Automation Project for Automation Exercise

## Overview
This repository documents a structured QA workflow for testing the public website [Automation Exercise](https://automationexercise.com).  
The project starts with deep manual testing, organized test case design, defect tracking, and traceability, then moves into Playwright + JavaScript automation.

The goal is to show a realistic software testing process instead of jumping directly into automation.

---

## Project Objectives
- Perform structured manual testing on a real public practice website
- Design reusable and maintainable test cases
- Track defects in a clear and professional format
- Build traceability between manual coverage and automation scope
- Automate high-value scenarios using Playwright with JavaScript
- Present the project as a continuous QA effort on GitHub

---

## Application Under Test
**Website:** Automation Exercise  
**URL:** https://automationexercise.com

### Core features covered
- Homepage validation
- Signup and login
- Products and product details
- Search functionality
- Cart behavior
- Checkout entry flow
- Contact Us form
- End-to-end user journeys

---

## Testing Approach

### Phase 1: Manual Testing
Manual testing was completed first to understand the product behavior, expected flows, validations, and edge cases.

This phase includes:
- Test planning
- Feature-wise test case design
- Smoke and regression tagging
- Priority assignment
- Bug report tracking
- Traceability preparation

### Phase 2: Automation Testing
After manual coverage was defined, selected scenarios were prepared for automation using:
- **Playwright**
- **JavaScript**
- **Page Object Model**
- **Reusable fixtures**
- **GitHub-based version history**
